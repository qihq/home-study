# 声音隐私与备份恢复

## 数据位置和责任边界

Compose 会将宿主机 `${FAMILY_LEARNING_DATA_DIR:-./data}` 同时挂载给 `app` 和 `worker` 的 `/data`。在 DS918+ 上，建议在 `.env` 中设置：

```dotenv
FAMILY_LEARNING_DATA_DIR=/volume1/family-learning
```

这个目录是应用的持久化边界，包含 `app.db`、上传和视频文件、TTS 缓存、参考声音、设置密钥及 `/data/backups/`。请把目录访问权限制给管理员和 Container Manager 所需账户；不要把它公开为匿名共享目录，也不要把其中的声音文件放到可被反向代理直接访问的 Web 目录。

## 声音样本和多人使用

声音样本只能在取得声音本人明确授权后录制或上传。界面要求勾选授权确认；录制样本需为 8-30 秒。应用会将样本标准化为 WAV，并使用它生成预览和朗读音频。声音克隆会将参考声音发送给所配置的 MiMo 服务处理，因此应在取得授权前说明这一点，并仅使用你信任且有权使用的服务账号。

每个“使用人”可有多个声音版本，其中一个已就绪版本可设为默认。朗读、词典音频和默写可选择不同的已就绪声音；切换不会覆盖既有默写记录中保存的使用人和声音名称快照。删除已被默写记录引用的声音版本时，应用会将其禁用而不是直接删除。

## `.flvoice` 声音包

声音包用于在部署间转移已就绪声音的参考 WAV 和声音元数据。包采用密码保护；导出包不包含 AI/TTS API Key，也不包含 TTS 生成缓存。

- 密码没有找回、重置或后门机制。遗失密码后，该 `.flvoice` 文件无法恢复；请使用密码管理器保存密码。
- 导入前反向代理须允许至少 50 MB 请求体。应用拒绝超过 50 MB 的上传包，并拒绝解密后内容总量超过 50 MB 的包。
- 为普通录制片段和声音样本上传，反向代理应至少允许 25 MB 请求体；当前应用没有单独的通用 25 MB 服务端拦截规则，这与声音包的 50 MB 应用上限是不同层级的配置。
- 导入过程会先预览，再选择合并、更新资料并合并，或新建使用人。导入完成后检查每个声音版本的状态和试听结果。

声音包包含可识别声音的参考音频。传输时使用 HTTPS，导出后仅存放在受控位置；不再需要的本地副本和下载副本应自行删除。

## 备份内容

“设置 - 数据备份”生成 `/data/backups/app-<UTC 时间戳>.zip`。该应用备份包含：

- `app.db`：SQLite 数据库；
- `reference-audio/`：数据库中已登记且仍存在于 `/data` 下的参考声音；
- `ai-settings.key` 与 `tts-settings.key`：用于解密已保存 AI/TTS API Key 的密钥文件（若存在）。

它不包含视频、普通上传文件、TTS 缓存、声音包下载文件，或未被数据库登记的文件。因此需要完整灾难恢复时，除创建应用备份外，还要用 Hyper Backup 备份整个 `FAMILY_LEARNING_DATA_DIR`，并保留历史版本。将 Hyper Backup 目标与 NAS 本机磁盘分离，才能应对 NAS 或磁盘故障。

## 升级前备份和迁移

升级镜像、修改 `FAMILY_LEARNING_DATA_DIR` 或运行可能触发数据库迁移的版本前：

1. 在设置页创建应用备份，确认 `/data/backups/` 中出现新的 `app-*.zip`。
2. 执行或确认 Hyper Backup 已成功备份整个宿主机数据目录，例如 `/volume1/family-learning`。
3. 记录当前镜像版本和 `.env` 中的数据目录，不要在备份验证前删除旧镜像或旧数据目录。
4. 再停止并更新 Compose 项目。

当前迁移在修改 SQLite 架构前还会写入 `/data/backups/pre-0002-*.db`。该文件只是一份迁移前数据库副本；恢复声音、密钥、视频和其他持久化文件仍要依赖完整目录备份或 `app-*.zip`。

## 恢复步骤

优先在隔离的测试目录验证恢复，确认无误后再替换生产数据。

### 恢复整个数据目录（推荐）

1. 停止 Compose 项目中的 `app` 和 `worker`，避免 SQLite 与文件在恢复时被写入。
2. 从 Hyper Backup 恢复整个 `FAMILY_LEARNING_DATA_DIR` 到原来的宿主机路径，例如 `/volume1/family-learning`。
3. 确认 `.env` 仍指向该路径，启动 Compose 项目。
4. 登录后核对视频、学习记录、声音版本、AI/TTS 设置和声音试听；必要时在设置页重试失败的声音处理任务。

### 仅恢复应用备份 ZIP

应用没有一键导入 ZIP 的接口。仅在完整目录备份不可用时按以下方式人工恢复：

1. 停止 `app` 和 `worker`，并先复制当前 `/data` 目录作为可回退副本。
2. 将 ZIP 解压到临时目录；用其中的 `app.db` 替换 `/data/app.db`，并把 `ai-settings.key`、`tts-settings.key` 放回 `/data/`。
3. 使用 `app.db` 中 `voice_versions.reference_audio_path` 的原始路径，将 `reference-audio/` 中同名的 WAV 放回对应的 `/data` 路径。ZIP 只保存声音文件名，不保存路径映射；同名文件或无法确定路径时，应停止并改用完整目录备份。
4. 启动项目，试听每个声音版本并检查 AI/TTS 设置。若密钥文件与数据库不成套，已保存的 API Key 可能无法解密，需要在设置页重新输入。

恢复后立即创建一份新的应用备份，并确认 Hyper Backup 的下次任务成功完成。
