# PWA 视频下载、辞典与发音可靠性设计

## 目标

修复从视频库保存 MP4 后 PWA 被下载地址替换、无法返回主页的问题；提供符合现有“动森小岛”视觉语言的应用内下载页和真实进度；下载完成后在支持的 iPhone/iPad 上自动唤起系统分享面板，让用户通过一次“存储视频”确认把文件加入照片图库。同时减少辞典和默写发音偶尔朗读提示词的问题，并让辞典结果展示更完整。

## 范围

本次包含四项彼此关联的用户体验改进：

1. 视频库不再用当前 PWA 窗口直接导航到文件地址。
2. 新增单任务、前台运行的应用内视频下载页。
3. 统一并收紧 MiMo 普通 TTS 与声音克隆的朗读请求，确保待朗读正文和控制提示严格分离。
4. 充分利用 ECDICT、CC-CEDICT 和 AI 已返回的数据，完整展示释义、词性、例句和用法。

不实现 Service Worker 后台下载、断点续传、下载队列或无需用户确认直接写入系统照片图库。iOS PWA 不提供静默写入照片图库的标准 Web API；应用只能在下载完成后唤起系统分享面板，由用户点击一次“存储视频”。

## 现状与根因

### 视频保存后无法返回

视频库目前使用带 `download` 属性的普通链接直接指向受认证的 `/api/recordings/{id}/download/720p`。在 iOS 独立模式 PWA 中，跨入媒体响应时 `download` 属性并不总会按桌面浏览器方式处理，当前应用窗口可能被文件响应或系统预览替换。录制完成页本身已有并测试过“返回主页”按钮，因此故障位于视频库的 MP4 保存链路，而不是录制完成页路由。

### 发音偶尔朗读提示词

MiMo 普通 TTS 和声音克隆都通过 chat-completions 消息传递控制提示与朗读正文。虽然当前提示要求只读 assistant 文本，但模型偶尔仍可能把 user 消息中的控制提示当成可朗读内容。需要让两类客户端共享同一套最小请求构造逻辑，并把可朗读正文限制为单独、经过规范化的字段；测试必须直接检查序列化请求，确保 UI 提示、翻译标签和其他控制文案不会混入正文。

### 辞典内容简略

本地 ECDICT 查询只把第一条翻译当作主释义、最多三条后续翻译当作备选，英文 definition 也只取第一条作为 usage note；前端结果卡又没有渲染 `alternatives`、`examples` 和 `usage_note`。因此即使后端已有内容，用户仍只看到很少一部分。CC-CEDICT 多条词条也需要按去重后的完整义项组合，而不是只突出第一条。

## 下载页设计

### 导航与状态

应用路由增加 `download` 屏幕和一个当前下载描述对象，至少包含录制 ID、标题、日期、语言和建议文件名。视频库的“保存 MP4”改为按钮，通过回调进入下载页；不再给当前 PWA 窗口设置媒体 URL。

下载页是 `AppShell` 内的独立页面，沿用现有小岛素材、圆角卡片、青绿色主色和黄色操作按钮。页面始终显示：

- 当前视频名称和类型；
- 真实百分比进度条；
- 已下载大小与总大小（服务器未提供总大小时显示已下载大小和不确定进度）；
- 当前阶段：准备、下载中、等待保存、已交给系统、失败或已取消；
- “取消下载”或“重试”；
- “返回视频库”和“返回主页”。

从下载页返回不会丢失服务器中的视频，也不会影响视频库。离开正在下载的页面时中止当前请求并释放内存中的 Blob URL。系统分享成功结束后，下载页显示短暂的“视频已保存，正在返回主页”确认状态，然后自动切换到主页，避免用户留在一个已经完成的任务页面。

### 下载数据流

下载页使用带 `credentials: include` 的 `fetch` 请求现有受认证下载端点。成功响应通过 `ReadableStream` 分块读取，以累计字节数除以 `Content-Length` 计算真实进度。后端 `FileResponse` 保持认证和文件存在校验，并由测试确认响应包含正确的 `Content-Length`、`Content-Type` 和附件文件名。

读取完成后把分块合并为 `video/mp4` Blob 和 `File`。实现需要处理以下状态：

- 响应失败：展示可读错误并允许重试；
- 流不可用：退回 `response.blob()`，显示不确定进度；
- 用户取消：通过 `AbortController` 停止请求并清空部分数据；
- 页面卸载：中止请求并释放所有对象 URL；
- 内存不足或分享失败：保留失败说明和手动保存回退操作。

### 保存到照片图库

文件准备完成后，若 `navigator.canShare({ files: [file] })` 为真，页面自动调用 `navigator.share({ files: [file] })`，打开系统分享面板。用户在 iOS 中点击“存储视频”即可加入照片图库。

浏览器要求分享调用必须直接来自用户手势时，自动调用可能被拒绝。此时页面进入“等待保存”状态，突出显示“保存到照片”按钮，由用户点击后再次调用分享 API。只有分享 Promise 正常完成时才自动返回主页；用户取消分享或分享失败不算下载失败，文件仍保持可保存状态，页面不会自动跳走。

若文件分享 API 不可用，则提供对象 URL 形式的“保存 MP4”回退按钮，并明确提示该设备需要通过浏览器下载或分享菜单手动保存。应用页面本身不会被媒体地址替换。

## 发音可靠性设计

新增一个小而独立的 MiMo 朗读请求构造函数，供普通 TTS 和声音克隆共同使用。它接收唯一的 `spoken_text` 和可选声音/参考音频参数，执行：

1. 去除首尾空白并拒绝空正文；
2. 保证控制提示只存在于控制消息中；
3. 保证 assistant 消息只包含 `spoken_text`，不拼接“请朗读”、翻译方向、词性或 UI 状态；
4. 使用更短、无示例内容的控制提示，避免模型把一段长提示当作朗读稿；
5. 让普通音色和克隆音色使用同一消息结构，避免两个路径继续漂移。

辞典发音仍只朗读英文：英译中读查询原文，中译英读主英文翻译。默写仍只读学习项正文。重新生成会使用新请求版本生成新资产，避免继续命中含提示词的旧缓存。

测试覆盖请求 JSON 中正文只出现于可朗读字段、控制提示不包含目标词或示例朗读文本，以及辞典与默写传入的正文没有 UI 标签。由于第三方模型仍有非确定性，不能从协议层承诺绝对零概率；最小化提示并统一消息结构可以消除应用自身混入提示词的路径。

## 辞典内容设计

### 本地数据整理

ECDICT 查询将全部非空翻译行按顺序去重，并解析常见的 `n.`、`v.`、`adj.` 等词性前缀生成多条 `parts_of_speech`。主释义采用第一条中文释义，其他不同义项进入扩充后的 `alternatives`。英文 definitions 按行整理为单独的英文释义/用法内容，不再只保留第一行。

CC-CEDICT 将同一简体/繁体词条的 definitions 按顺序去重，保留拼音，并把首义作为主译、其余义项作为备选。明显属于量词、姓氏、参见说明等内容仍保留，但以分项方式展示，不混成一个长字符串。

现有 `alternatives` 最多三条的 schema 限制会提高到一个适合家庭阅读的有限上限，防止极端词条撑爆页面，同时不再把常见多义词截得只剩一两个意思。缓存提示版本升级，确保旧的简略结果不会继续命中。

### 结果卡展示

结果卡按以下顺序显示：

1. 查询原文、主翻译和音标/拼音；
2. 分词性的释义；
3. “其他释义”；
4. “英文释义 / 用法提示”；
5. 双语例句；
6. 发音、重新生成和标记生词操作。

空区块不渲染。来源标识与授权说明继续保留。AI 辞典提示词会明确要求对单词返回常用词性、多条常用义项、简洁用法说明和至少一个自然双语例句；短语与句子则优先给准确翻译和必要的用法说明，避免为凑数量生成低质量内容。

## 测试与验收

### 自动化测试

- 视频库点击保存时调用应用内下载回调，不存在会替换 PWA 页面的媒体链接。
- 下载器根据分块和 `Content-Length` 更新真实百分比，支持取消、失败重试和无流回退。
- 下载完成后优先分享 `video/mp4` 文件；自动分享被手势策略拒绝时显示手动“保存到照片”。
- 系统分享成功结束后自动返回主页；分享取消或失败时留在下载页，允许再次保存。
- 下载页无论成功或失败都能手动返回视频库和主页，并在卸载时取消请求、释放 URL。
- 下载 API 保持登录保护并返回进度计算需要的响应头。
- MiMo 两条 TTS 路径共享请求构造规则，正文不会混入提示、标签或翻译说明。
- 辞典本地查询保留多词性、多义项和多行英文释义。
- 结果卡显示 alternatives、examples 与 usage note，且空内容不显示空标题。
- 既有录制、视频库、默写、辞典和 TTS 测试全部通过。

### 手动验收

在安装到主屏幕的 iPhone/iPad PWA 中打开视频库，点击“保存 MP4”，确认进入小岛下载页并看到进度变化。下载完成后系统分享面板自动出现；选择“存储视频”，确认视频出现在照片图库且 PWA 自动回到主页。再次下载并取消系统分享，确认 PWA 留在下载页，可再次保存或手动返回视频库/主页，并且应用没有被媒体预览替换。

分别用默认 MiMo 音色和克隆音色多次播放单词、短语与辞典翻译，确认只朗读目标正文。查询一个常见多义英文单词和一个多义中文词，确认主释义、其他释义、词性、英文说明和可用例句均清晰显示。

## 兼容性与限制

- iOS 不允许 PWA 静默写入照片图库，必须由用户在系统分享面板确认“存储视频”。
- iOS 对 Web Share 的自动调用可能要求用户手势，因此必须保留显眼的手动保存按钮。
- 下载 Blob 驻留内存，超大视频可能受设备内存限制；本次使用现有 720p 压缩文件，并对内存或分享错误给出恢复提示。
- PWA 退到后台后浏览器可能暂停网络和 JavaScript，进度页只承诺前台可靠运行，不宣称后台下载。
